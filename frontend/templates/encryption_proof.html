What is this page?

This page provides visual proof that sensitive health record data is genuinely encrypted at rest. The analysis below shows the raw encrypted bytes stored in the database alongside the decrypted plaintext — proving that no plaintext clinical data is ever stored.

Proof:

The 240-byte hex string above is what the database stores. The plaintext above is reconstructed by decrypting that ciphertext using the application-layer key and the extracted IV. A database admin sees only the hex — never the clinical text.

Security Statement

Each field uses an independently generated random IV, so even identical values produce different ciphertext.

Database administrators see only binary ciphertext — never plaintext clinical data.

The encryption key is stored in the application environment — never in the codebase.
