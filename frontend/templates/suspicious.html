About the Detection Engine

Rule 1 — Rapid Record Access: Triggered when a user views more than 10 records within a 2-minute window. May indicate data exfiltration.

Rule 2 — New IP Login: Triggered when a user logs in from an IP address not previously associated with their account. May indicate credential compromise.

Rule 3 — Brute Force: Triggered after 3+ consecutive failed login attempts from the same IP. Account is automatically locked per security policy.

Rule 4 — Unauthorized Access: Triggered when any access attempt is made to a route that the user's role does not permit. HTTP 403 is returned and the attempt is logged.

This detection engine uses rule-based logic only — no machine learning required. All events are logged in the audit trail with full metadata.
